# Voice Gender Mismatch Fix

## Problem Summary
The application was crashing with the error:
```
google.api_core.exceptions.InvalidArgument: 400 Requested female voice, but voice en-IN-Neural2-C is a male voice.
```

## Root Cause Analysis
The issue was caused by a **hardcoded voice mapping** in the TTS model that was overriding all configuration files. The system was:
1. Requesting `ssml_gender: "FEMALE"`
2. But using `en-IN-Neural2-C` which is actually a **MALE** voice

## Files Fixed

### 1. Core Issue - Hardcoded Voice Map
**File**: `june/june_va/models/tts.py` (Line 88-97)

**Changed**:
```python
# BEFORE (WRONG - Male voices)
"EN-IN": "en-IN-Neural2-C",     # Indian English female ❌ WRONG!
"EN-US": "en-US-Neural2-C",     # US English female ❌ WRONG!
"EN": "en-IN-Neural2-C",        # Default English to Indian ❌ WRONG!

# AFTER (CORRECT - Female voices)
"EN-IN": "en-IN-Neural2-D",     # Indian English female ✅ CORRECT!
"EN-US": "en-US-Neural2-F",     # US English female ✅ CORRECT!
"EN": "en-IN-Neural2-D",        # Default English to Indian ✅ CORRECT!
```

### 2. Configuration Files Fixed
All configuration files were also updated (though the hardcoded map was overriding them):

- `june/config.json`
- `june/config-enhanced-multilingual.json`
- `june/gcloud-multilang-config.json`

Changed all instances of `en-IN-Neural2-C` → `en-IN-Neural2-D`

## Google Cloud TTS Voice Reference

### Indian English Voices
- **Male**: `en-IN-Neural2-A`, `en-IN-Neural2-B`, `en-IN-Neural2-C`
- **Female**: `en-IN-Neural2-D`, `en-IN-Wavenet-D`, `en-IN-Standard-D`

### US English Voices
- **Male**: `en-US-Neural2-A`, `en-US-Neural2-D`, `en-US-Neural2-I`, `en-US-Neural2-J`
- **Female**: `en-US-Neural2-C`, `en-US-Neural2-E`, `en-US-Neural2-F`, `en-US-Neural2-G`, `en-US-Neural2-H`

### Hindi Voices
- **Male**: `hi-IN-Neural2-B`, `hi-IN-Neural2-C`, `hi-IN-Neural2-D`
- **Female**: `hi-IN-Neural2-A`, `hi-IN-Wavenet-A`, `hi-IN-Standard-A`

## Why This Happened
The comments in the code said "Indian English female" next to `en-IN-Neural2-C`, but this was **incorrect**. According to Google Cloud's documentation:
- `en-IN-Neural2-C` is actually a **MALE** voice
- `en-IN-Neural2-D` is the correct **FEMALE** voice
