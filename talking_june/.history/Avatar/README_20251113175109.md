## 📦 Quick Start

### 1. Prerequisites

- **Python 3.8+** installed
- **Ollama** with llama3.2 model
- **Google Cloud TTS API** (optional, for high-quality voices)
- Modern web browser (Chrome, Firefox, Edge)

### 2. Install Dependencies

```powershell
cd june
pip install -r requirements.txt
```

### 3. Start Everything (Automated)

**Option A - PowerShell Script (Recommended):**
```powershell
.\start_all.ps1
```

**Option B - Manual (3 Terminals):**

Terminal 1:
```powershell
python bridge_server.py
```

Terminal 2:
```powershell
cd TalkingHead
python -m http.server 8080
```

Terminal 3:
```powershell
cd june
python -m june_va
```

### 4. Open Browser

Navigate to: **http://localhost:8080**

### 5. Start Talking! 🎉

Speak to June and watch the avatar lip-sync in real-time!

## 🔧 Configuration

### June VA Configuration (`june/config.json`)

```json
{
  "llm": {
    "model": "llama3.2:latest",
    "disable_chat_history": false
  },
  "tts": {
    "model": "google",
    "generation_args": {
      "credentials": "path/to/service-account.json"
    }
  },
  "stt": {
    "model": "google",
    "generation_args": {
      "credentials": "path/to/service-account.json"
    }
  }
}
```

### Bridge Server Configuration

Default ports (in `bridge_server.py`):
- **WebSocket:** 8765
- **HTTP Audio:** 8001
- **TalkingHead:** 8080

## 🎓 How It Works

1. **User speaks** → June's STT converts to text
2. **LLM processes** → Ollama generates response
3. **TTS generates** → Audio file created in `shared_audio/`
4. **WebSocket notifies** → Bridge sends URL to browser
5. **Avatar speaks** → TalkingHead plays audio with lip-sync

## 🐛 Troubleshooting

### Common Issues

| Issue | Solution |
|-------|----------|
| Bridge not connected | Ensure `bridge_server.py` is running first |
| Audio not playing | Check browser console for CORS errors |
| Port in use | Close other services or change ports |
| No lip-sync | Verify WebSocket connection in console |

### Check Connection Status

**Browser Console:**
```javascript
// Should show: 1 (OPEN)
console.log(window.juneWebSocket?.readyState);

// Manual reconnect if needed
window.reconnectJuneBridge();
```

**Bridge Server:**
```
[WS] ✅ Client connected: 127.0.0.1:xxxxx (Total: 1)
```

### View Logs

All logs are color-coded:

- 🟡 **Yellow** - Warnings (usually safe to ignore)
- 🔴 **Red** - Errors (need attention)

## 🚀 Performance Tips

1. **Use SSD** - Store audio files on SSD for faster I/O
2. **Close unused apps** - Free RAM for better LLM performance
3. **Wired connection** - Reduce latency with Ethernet
4. **GPU acceleration** - Enable CUDA for faster TTS (if available)

## 🔐 Security Notes

- Bridge server binds to `localhost` only (not exposed to network)
- Audio files are temporary and cleaned up automatically
- No API keys are transmitted over WebSocket
- CORS is enabled only for local development
